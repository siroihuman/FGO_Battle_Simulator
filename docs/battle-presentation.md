# 戦闘中のゲージ表示とコマンドカード識別

この表示処理は次のファイルで管理します。

- `js/battle-presentation.js`
- `js/battle-presentation-overlay.js`
- `js/app.js`
- `css/battle-presentation.css`
- `css/battle-presentation-overlay.css`

戦闘計算自体は`engine.js`の既存処理を使用し、表示用ランタイムは計算結果を変更しません。

## コマンドカード選択時のスクロール固定

コマンドカード、宝具、敵対象を選択して画面を再描画する前に、次の位置を保存します。

- ページの縦・横スクロール位置
- 味方・敵・コマンドカード・宝具・スキル欄の横スクロール位置

再描画直後と次の描画フレームで同じ位置へ戻します。マウス操作ではボタン押下時の自動フォーカスも抑制します。

## 攻撃開始後の全画面ターン表示

「攻撃開始」を押すと、通常の戦闘画面全体を覆う固定オーバーレイを新規表示します。コマンドカード欄の高さには依存しません。

表示内容：

- 攻撃開始時点の敵全体のHPとHPゲージ
- 攻撃開始時点の味方全体のHP・NPと各ゲージ
- 処理中のWave番号
- 攻撃・ターン処理中であることを示す表示

デスクトップでは敵と味方を左右に並べ、味方6騎を複数列で表示します。画面幅が狭い場合は縦配置へ切り替え、全画面内をスクロールできます。

ターン開始前と処理後の値を保存し、約0.9秒かけて数値とゲージ幅を補間します。OS・ブラウザで「視差効果を減らす」が有効な場合は短いアニメーションへ切り替えます。

## 3Wave時の敵HP表示

エンジンは各Waveで敵IDとして`enemy-1`～`enemy-3`を再利用します。そのため、ID一致だけで前後状態を対応付けると、突破した旧Waveの敵を次Waveの新しい敵として誤認します。

表示用スナップショットには各ユニットのWave番号も保存します。攻撃前後でWave番号が変わった場合、同じ敵IDが次Waveに存在していても、攻撃開始前の敵は次の状態をアニメーション目標として扱います。

```text
HP 0
戦闘不能
```

これにより次の両方で、突破したWaveの敵HPが0まで減少する様子を表示します。

- WAVE 1 → WAVE 2
- WAVE 2 → WAVE 3

見出しにも`WAVE 1/3 COMPLETE → WAVE 2/3`のように旧Waveと新Waveを表示します。

## アニメーション終了後の操作

アニメーション中は全画面オーバーレイを閉じられず、背後の戦闘画面も操作できません。

アニメーションと内部ターン遷移の完了後、次の案内を表示します。

```text
画面をクリック／タップして閉じる
```

この時点から全画面にクリック・タップ判定を設定します。アニメーション中から押し続けていた操作では閉じず、完了後に開始した新しいクリックまたはタップだけを受け付けます。EnterまたはSpaceでも閉じられます。

閉じた後の表示：

- 戦闘継続中：次ターンまたは次Waveのスキル・宝具・コマンドカード選択画面
- 勝敗確定時：戦闘結果画面

クリックまたはタップされるまでは、オーバーレイを自動では消去しません。

## 同一サーヴァントの識別

同じ`servantId`のサーヴァントが2騎以上編成されている場合、編成枠順に次の識別表示を付けます。

```text
同名1・前衛1
同名2・前衛3
同名3・控え2
```

識別表示は次の場所で共通して表示します。

- 味方ステータス欄
- コマンドカード
- 宝具カード
- 味方対象選択
- オーダーチェンジ対象選択

識別番号ごとに左枠とバッジの色を変えます。コマンドカードまたは宝具を選択すると、対応する味方ステータス欄も強調します。

## 回帰テスト

`tests/battle-presentation-tests.js`と`tests/battle-presentation-overlay-tests.js`で次を検証します。

- 同一サーヴァントの編成順識別
- 単独編成時に識別表示を付けないこと
- HP・NPスナップショット
- ゲージ補間値
- WAVE 1→2とWAVE 2→3の見出し
- Wave移行時に同じ敵IDが再利用されても旧敵をHP0にすること
- 全画面固定オーバーレイ
- アニメーション完了後だけクリック・タップで閉じる処理
- 旧コマンド欄内アニメーションの非表示
