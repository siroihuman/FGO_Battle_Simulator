# 戦闘中のゲージ表示とコマンドカード識別

この表示処理は次のファイルで管理します。

- `js/battle-presentation.js`
- `js/app.js`
- `css/battle-presentation.css`

戦闘計算自体は`engine.js`の既存処理を使用し、表示用ランタイムは計算結果を変更しません。

## コマンドカード選択時のスクロール固定

コマンドカード、宝具、敵対象を選択して画面を再描画する前に、次の位置を保存します。

- ページの縦・横スクロール位置
- 味方・敵・コマンドカード・宝具・スキル欄の横スクロール位置

再描画直後と次の描画フレームで同じ位置へ戻します。マウス操作ではボタン押下時の自動フォーカスも抑制します。

## 攻撃開始後のターン表示

「攻撃開始」を押すと、現在のコマンド欄を同じ高さのターン処理表示へ切り替えます。

表示内容：

- 敵全体のHPとHPゲージ
- 味方全体のHP・NPと各ゲージ
- 攻撃・ターン処理中であることを示す表示

ターン開始前と処理後の値を`battle-presentation.js`で保存し、約0.9秒かけて数値とゲージ幅を補間します。OS・ブラウザで「視差効果を減らす」が有効な場合は短いアニメーションへ切り替えます。

Waveを突破した場合、旧Waveの敵はHP0まで減少したことを表示してから次Waveへ切り替えます。

## 次ターンへの自動移行

ゲージアニメーション完了後は追加のクリックやタップを要求せず、次の状態へ自動的に移行します。

- 戦闘継続中：次ターンのスキル・宝具・コマンドカード選択画面
- 勝敗確定時：戦闘結果画面

ページの縦スクロール位置は攻撃開始時の位置を維持します。

## 同一サーヴァントの識別

同じ`servantId`のサーヴァントが2騎以上編成されている場合、編成枠順に次の識別表示を付けます。

```text
同名1・前衛1
同名2・前衛3
同名3・控え2
```

識別表示は次の場所で共通して表示します。

- 味方ステータス欄
- コマンドカード
- 宝具カード
- 味方対象選択
- オーダーチェンジ対象選択

識別番号ごとに左枠とバッジの色を変えます。コマンドカードまたは宝具を選択すると、対応する味方ステータス欄も強調します。

## 回帰テスト

`tests/battle-presentation-tests.js`で次を検証します。

- 同一サーヴァントの編成順識別
- 単独編成時に識別表示を付けないこと
- HP・NPスナップショット
- ゲージ補間値
- Wave移行時の旧敵HP0表示
- スクロール固定、ゲージ表示、自動次ターン遷移の登録
