# クラス相性

## 基本方針

攻撃時のクラス相性倍率は、`js/class-affinity-rules.js`で一元管理する。

- 有利：2倍または1.5倍
- 不利：0.5倍
- 等倍：1倍
- シールダーは攻撃・防御ともに全クラスへ等倍

ダメージ計算と宝具・コマンドカード上の`WEAK`／`RESIST`表示は、同じ`resolveAttackClassAffinity`を使用する。

## 通常クラス

以下の16クラスについて、AppMediaの攻撃側視点の相性表を登録している。

- セイバー
- アーチャー
- ランサー
- ライダー
- キャスター
- アサシン
- バーサーカー
- ルーラー
- アヴェンジャー
- ムーンキャンサー
- アルターエゴ
- フォーリナー
- プリテンダー
- ビースト（ドラコー）
- ビースト（スペース・エレシュキガル）
- ビースト（U-オルガマリー）

参照資料：<https://appmedia.jp/fategrandorder/768578>

## 固有ビースト

`classId: beast`は、通常のクラス同士では等倍とし、攻撃者・防御者ごとの固有プロファイルで相性を決定する。

### ルルイエ

| 判定 | 対象 |
|---|---|
| 1.5倍有利 | 〔ヒト科〕または〔今を生きる人類〕 |
| 0.5倍不利 | なし |
| 等倍 | その他すべて |

### ────

| 判定 | 対象 |
|---|---|
| 1.5倍有利 | 〔人の力〕または〔ヒト科〕 |
| 0.5倍不利 | 〔天の力〕または〔ヒト科以外〕 |
| 等倍 | その他すべて |

`〔ヒト科以外〕`は「ヒト科を持たない全対象」という否定条件ではなく、シミュレーターへ明示的に登録された特性として判定する。したがって、〔ヒト科〕と〔ヒト科以外〕のどちらも持たない対象は、天地条件にも該当しなければ等倍になる。

有利条件と不利条件が同時に成立する場合は、有利条件を優先する。

例：〔天の力〕かつ〔ヒト科〕は1.5倍有利。

未登録サーヴァントとして追加する場合は、データに`classAffinityProfile: 'firstMurder'`を指定するか、表示名を`────`とする。

### バフォメット

#### 攻撃相性

| 判定 | 対象 |
|---|---|
| 2倍有利 | キャスター、ルーラー |
| 1.5倍有利 | クラススキル〔道具作成〕、クラススキル〔陣地作成〕、または〔悪魔〕 |
| 0.5倍不利 | なし |
| 等倍 | その他すべて |

対象がキャスターまたはルーラーで、同時に条件特性・クラススキルも持つ場合は2倍有利を優先する。

クラススキルはサーヴァントデータの`passives[].name`で判定する。敵設定などクラススキルデータを持たない対象では、特性欄へ`道具作成`または`陣地作成`を記述した場合も同じ条件として扱う。

#### 防御相性

| 判定 | 攻撃側 |
|---|---|
| 2倍防御不利 | アヴェンジャー、ビースト |
| 等倍 | その他すべて |

ビーストには汎用`beast`に加え、ドラコー、スペース・エレシュキガル、U-オルガマリーの個別ビーストクラスIDを含む。

## 旧固有処理との統合

ルルイエとバフォメットの固有ファイルには、以前の2倍固定相性処理が残っている。完全版相性ルールを固有ファイルより後に読み込み、計算中だけ旧固有判定を抑止することで、最終倍率を`class-affinity-rules.js`へ一元化する。

計算後はクラスID、サーヴァントID、相性プロファイルを必ず元へ戻す。

## 表示

宝具・コマンドカードは、現在選択中のエネミーに対して次を表示する。

- 倍率が1より大きい：`WEAK`
- 倍率が1より小さい：`RESIST`
- 倍率が1：表示なし

`WEAK`／`RESIST`はカード上部中央へ表示し、左上のサーヴァント名欄と右上の選択順表示を避ける。特性付与や対象変更によって相性が変化した場合は、画面再描画時に再判定する。
