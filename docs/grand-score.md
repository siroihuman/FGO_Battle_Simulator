# グランドスコアシステム

参照資料：<https://appmedia.jp/fategrandorder/78904553>

## 基本方針

- グランドスコアの効果量は常に最大値として扱う。
- 編成設定画面の「グランドスコアシステムを使用」でシステム全体をON／OFFする。
- 各編成枠の「グランドサーヴァント」で適用対象を個別に指定する。
- グランドサーヴァントにクラスごと1騎などの騎数制限は設けない。同一クラスを含む複数騎へ同時に適用できる。
- 控えのグランドサーヴァントも指定状態を保持し、前衛へ登場した時点から前衛限定効果を使用する。

## グランドサーヴァント能力値

グランドサーヴァントはHPとATKをそれぞれ1,000増加する。

この増加は戦闘中の強化状態ではなく、フォウ強化と同じ育成値として扱う。

```text
最終HP  = 基礎HP + 通常HPフォウ + グランドHPフォウ1,000 + 概念礼装HP
最終ATK = 基礎ATK + 通常ATKフォウ + グランドATKフォウ1,000 + 概念礼装ATK
```

したがって、強化解除・強化無効・強化解除耐性の対象にはならない。

## 全クラス共通効果（最大値）

| 効果 | 最大値 |
|---|---:|
| Quickカード性能アップ | 50% |
| Artsカード性能アップ | 50% |
| Busterカード性能アップ | 50% |
| Extra Attackカード性能アップ | 200% |
| 開始時NP | 10% |
| 毎ターンNP | 10% |
| HP回復量アップ | 50% |
| 強化解除耐性 | 80% |
| マスタースキルの一部の効果アップ | 50% |
| マスタースキルCT進行 | 通常の2倍 |

共通効果は`sourceType: 'grandScore'`を持つ解除不能状態として管理する。HP／ATK+1,000だけは状態ではなくフォウ値へ加算する。

## マスタースキル効果アップ

倍率を持つ数値効果だけを1.5倍にする。継続ターン数、付与回数、スキルチャージ短縮数などは変更しない。

```text
HP3,000回復       → HP4,500回復
攻撃力50%アップ   → 攻撃力75%アップ
無敵1ターン       → 無敵1ターンのまま
スキルCTを2短縮   → 2短縮のまま
```

対象例：回復量、攻撃力、防御力、カード性能、宝具威力、クリティカル威力、NP増加量、スター獲得数など。

対象外例：`cooldownReduce`、無敵・回避・ガッツなどの回数、状態の継続ターン数。

グランドサーヴァント本人が回復対象の場合、マスタースキル数値効果アップ後の回復量へ、共通のHP回復量50%アップも適用する。

## 実装ファイル

- `js/grand-score-core.js`
  - 編成指定の解決
  - フォウ加算
  - 共通効果
  - マスタースキル補正
  - 毎ターン処理
- `js/grand-score-chain.js`
  - Quick／Arts／Buster Chain
  - Mighty Chain
  - Brave Chain
  - Chain不成立
- `js/grand-score-combat.js`
  - 攻撃前後
  - 宝具前後
  - 被攻撃
  - ガッツ
  - ターゲット固定
  - 弾数・特殊状態
- `js/grand-score-ui.js`
  - 編成設定画面の全体ON／OFF
  - 編成枠ごとのグランド指定
- `css/grand-score.css`
  - 編成画面表示

## サーヴァント固有例外との境界

グランドスコアの通常処理・限定的例外処理はシステム側で管理する。サーヴァント固有ファイルからグランドスコアを直接判定しない。

サーヴァント固有の宝具換装、カード種別変更、特殊な攻撃差し替えなどがある場合も、最終的な実効カード種別・攻撃種別を共通ランタイムへ渡してからグランドスコアを判定する。今後のサーヴァント追加時は、以下を回帰確認する。

- 宝具カード種別変更後のチェイン判定
- 行動不能によるチェインエラーとの優先順位
- 攻撃差し替え時の攻撃前後トリガー
- 宝具前・宝具後効果の実行順
- オーダーチェンジ／死亡交代時の前衛限定効果
- 強化解除不能状態の保持
