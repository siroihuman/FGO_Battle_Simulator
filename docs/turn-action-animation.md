# ターン行動タイムラインと順次攻撃表示

この機能は次の共通ファイルで管理します。

- `js/turn-action-timeline.js`
- `js/battle-presentation.js`
- `js/battle-presentation-overlay.js`
- `js/star-cap-99.js`
- `css/battle-action-sequence.css`

サーヴァントIDや名称による分岐は使用しません。

## 行動順

攻撃開始後、実際にエンジンが処理した行動を次の順番で記録します。

1. 味方攻撃フェーズ
   - コマンドカード1
   - コマンドカード2
   - コマンドカード3
   - Brave Chain成立時はExtra Attack
2. 敵攻撃フェーズ
   - エネミー1
   - エネミー2
   - エネミー3

戦闘不能や行動不能により実行されなかった行動は、エンジンの実行結果に従います。敵が行動不能の場合も、その敵の順番と行動不能表示はタイムラインに残します。

## 行動ごとのスナップショット

各行動について次の2点を保存します。

- 行動開始直前
- 行動処理完了直後

スナップショットには以下を含みます。

- 味方・敵のHP
- 味方のNP
- 現在Wave・Turn・Phase
- 現在スター
- 次ターン獲得予定スター

全画面表示はターン全体の開始値と終了値を一度に補間せず、各行動の前後値を順番に補間します。

## 行動者の発光

各行動開始時、行動者に対応する表示枠へ`acting`クラスを一時付与します。

- 味方：金色の発光
- 敵：赤色の発光
- 発光後にHP・NP・スターの変化を再生

OSやブラウザで「視差効果を減らす」が有効な場合は発光とゲージ変化を短縮します。

## スター獲得表示

全画面表示の右上に次を表示します。

```text
獲得スター ★ 0/99
```

カード・宝具・Extra Attack・Quick Chain・トリガー効果による次ターンスターを累積表示します。

## スター上限

スター所持数と次ターン獲得数の上限は99個です。

- 初期スター：0～99
- コマンドフェイズ中の即時スター獲得：最大99
- 攻撃中の次ターンスター獲得：最大99
- ターン更新後のスター：最大99

コマンドカードは5枚で、各カードのクリティカル率に使用できるスターは最大10個のため、カードへの配分処理だけは従来どおり最大50個です。51個以上を所持していても表示値は保持されますが、配分対象は50個までです。

## 回帰テスト

- `tests/turn-action-timeline-tests.js`
- `tests/star-cap-99-tests.js`
- `tests/battle-action-sequence-tests.js`

以下を検証します。

- 味方1→2→3、敵1→2→3の順序
- 行動ごとの前後スナップショット
- 累積スター獲得表示
- 行動者枠の発光登録
- 味方・敵フェーズ表示
- スター99個上限
- クリティカル配分は最大50個
