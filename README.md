## 敵特性の入力

敵設定の「特性（記述式）」欄へ次のように入力します。

```text
神性 / 王 / 人型
```

区切りには `/`、`／`、カンマ、改行を使用できます。効果コード側でも同じ文字列を使います。

```js
{ type:'traitPowerUp', target:'self', trait:'王', value:50, duration:3 }
```

詳細：`templates/ENEMY_TRAIT_GUIDE.md`

## スマートフォンで使用する方法

画面はスマートフォン幅へ対応しています。ただし、スマートフォンではZIP内の`index.html`を直接開く方法が端末やブラウザによって不安定です。確実な方法は、フォルダー一式をWebサーバーへ置き、HTTPSのURLから開くことです。

利用しやすい方法：

1. GitHub Pagesへ公開する
2. Netlify Dropへフォルダーをアップロードする
3. PCとスマートフォンを同じWi-Fiへ接続し、PCでローカルWebサーバーを起動する

セーブ設定はブラウザのLocalStorageへ保存されるため、別端末・別ブラウザ間では自動共有されません。

## ダメージ計算の実装仕様（v1.2.2）

ダメージは次の区分で計算します。

1. **A枠**：ATK、宝具倍率、カード補正、カード性能、クラス補正、クラス相性、天地人相性、乱数、攻撃補正0.23
2. **B枠**：攻撃力・防御力、クリティカル、Extra、特殊耐性
3. **C枠**：特攻状態、被ダメージ増減、クリティカル威力、宝具威力を加算
4. **D枠**：宝具固有の〔特性〕特攻倍率を乗算
5. **E枠**：固定与ダメージと固定被ダメージを最後に加算

乱数補正は **0.900～1.099** の0.001刻みです。

### 属性特攻の入力について

敵設定で属性を「人」にした場合、内部では自動的に〔人の力〕としても扱います。
そのため、特性欄へ改めて「人の力」と入力する必要はありません。
同様に「天」「地」「星」「獣」も、それぞれ〔天の力〕〔地の力〕〔星の力〕〔獣の力〕として判定されます。

### 回帰テスト

以下を実行すると、フェンリルの基準ダメージを検証できます。

```text
node tests/damage-regression.js
```

検証条件：フェンリルLv.120、ATKフォウ+2000、宝具Lv.5、Buster性能119%、〔神性〕特攻30%、〔人の力〕特攻100%、与ダメージ+95、最低乱数0.900。期待値は **242,876** です。


## 多段Hit攻撃の戦闘ログ表示

v1.2.3以降、宝具や通常攻撃が複数Hitする場合、敵が途中Hitで戦闘不能になっても、戦闘ログには全Hit分の総ダメージを表示します。

例として5Hit宝具で敵が3Hit目に戦闘不能になった場合でも、3Hit目までの累計値ではなく、4Hit目・5Hit目を含む宝具全体の総ダメージが表示されます。

内部では次の2つを分けて管理しています。

- `damage`：全Hit分の理論総ダメージ。戦闘ログに使用します。
- `actualHpDamage`：敵HPから実際に減少した量。HP処理用です。
